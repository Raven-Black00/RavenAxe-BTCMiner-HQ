  ᛒᚢᚷᚠᛁᛉᛖᛊ  —  BUG FIXES
  ─────────────────────────────────────────────────────────────────────────────

  [FIX-001]  ᛊᛖᛏᛏᛁᚾᚷᛊ ᛞᛁᚡᛁᛞᛖᚱ  —  Settings Window Center Divider
             SEVERITY : Visual / Medium
             LOCATION : SettingsWindow — two-column layout builder
             SYMPTOM  : The purple vertical divider between the left and right
                        settings columns was never rendered. It was placed using
                        cols_frame.winfo_reqwidth()//2 as a padx offset, but
                        winfo_reqwidth() always returns 0 at widget build time,
                        so the frame had zero offset and zero width on screen.
             FIX      : Introduced a dedicated divider column (index 1) in the
                        grid. Left content stays at column 0; right content
                        moved to column 2. Divider frame placed at column 1
                        with sticky="ns". No dynamic width calculation needed.

  [FIX-002]  ᛒᛚᛟᚲ ᚠᛟᚢᚾᛞ ᚨᛚᛖᚱᛏ  —  Block Found Discord Alert Never Fired
             SEVERITY : Functional / High  ← Discord alert was SILENT on blocks
             LOCATION : RavenMinerDash.update_display()
             SYMPTOM  : The ⛏️ BLOCK FOUND Discord alert condition evaluated
                        new_blocks > self.found_blocks, but self.found_blocks
                        had already been overwritten with new_blocks earlier in
                        the same method. The comparison was always False.
                        The Valknut flash still triggered correctly (it used
                        the comparison before the assignment) but the Discord
                        webhook was permanently silenced.
             FIX      : Captured old_blocks = self.found_blocks before the
                        assignment. All downstream comparisons now use
                        old_blocks. Also corrected the initial guard condition
                        from self.found_blocks > 0 to old_blocks >= 0 so the
                        very first block is also announced.

  [FIX-003]  ᚠᛖᛏᚲᚺ ᚱᚨᚲᛖ ᚲᛟᚾᛞᛁᛏᛁᛟᚾ  —  fetch_miner() Global Snapshot Race
             SEVERITY : Stability / Low-Medium
             LOCATION : RavenMinerDash.fetch_miner()
             SYMPTOM  : fetch_miner() read the module-level globals API_URL and
                        MINER_IP directly. If the user changed the miner IP in
                        the Settings window while a poll was in-flight, the
                        offline alert message could reference the NEW IP rather
                        than the one that actually timed out, producing a
                        misleading "lost contact with X" message for the wrong
                        host.
             FIX      : Both globals are now snapshotted into local variables
                        _url and _ip at the very start of fetch_miner(). Each
                        poll is fully self-contained and atomic with respect to
                        mid-call IP changes.

  [FIX-004]  ᚱᛖᚠᚱᛖᛊᚺ ᛊᛚᛖᛖᛈ  —  CPU Spin on REFRESH = 0
             SEVERITY : Performance / Low
             LOCATION : RavenMinerDash.refresh_loop()
             SYMPTOM  : If the user set the refresh interval to 0 (or a value
                        below 0.1) in the settings entry, time.sleep(0) would
                        cause the background polling thread to spin at 100%
                        CPU utilisation. The BTC counter guard already used
                        max(REFRESH, 0.1) but the sleep() call did not.
             FIX      : time.sleep() now uses max(REFRESH, 0.1), matching the
                        existing BTC counter guard and enforcing a floor of
                        100ms between polls regardless of user input.

  Initial release candidate of RavenMiner HQ 3.9.0.
  Full-featured real-time dashboard for AxeOS / BitAxe ASIC miners.

  ᚠᛖᚨᛏᚢᚱᛖᛊ  —  FEATURES
  ─────────────────────────────────────────────────────────────────────────────
  · Real-time API polling (configurable interval, default 0.5s)
  · Three-column layout: Temperatures | Hashrate | Power & Hardware
  · Animated Vegvisir compass (procedural canvas drawing)
  · Valknut alpha-fade animation (16-frame flash on share accepted / block found)
  · Dual temperature gauges: ASIC + VR with colour-coded needle
  · Hashrate history bar chart (last 20 samples)
  · BTC price ticker via CoinGecko API (refreshed every ~60s)
  · Discord webhook alerts: overheat, VR overheat, low hashrate, block found,
    rejected share spike, and miner offline (3 consecutive failed polls)
  · System tray minimise support (pystray, optional)
  · First-run IP setup dialog with config persistence (ravenminer_config.json)
  · In-app Settings window with live miner config PATCH support
  · Gear icon + rune label click-to-settings with hover animation
  · ESC to quit keybinding
  · PyInstaller-aware path resolution for frozen .exe deployment
